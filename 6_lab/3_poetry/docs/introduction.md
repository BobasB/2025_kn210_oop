# Вступ до віртуальних середовищ

## Що таке віртуальне середовище?

**Віртуальне середовище** (virtual environment) - це ізольоване Python оточення, яке дозволяє встановлювати пакети для конкретного проекту без впливу на інші проекти або системний Python.

## Навіщо потрібні віртуальні середовища?

### 🔒 Ізоляція залежностей

Кожен проект може мати свої власні версії бібліотек без конфліктів:

```
Проект A: Flask 2.0, requests 2.28
Проект B: Flask 3.1, requests 2.32
```

### 🚀 Легка відтворюваність

Завдяки файлам залежностей (`requirements.txt`, `Pipfile`, `pyproject.toml`) можна легко відтворити середовище на іншій машині.

### 🛡️ Безпека системи

Встановлення пакетів у віртуальному середовищі не впливає на системний Python.

### 📦 Чистота проекту

Легко бачити, які саме пакети використовує проект.

## Типи інструментів

### venv (вбудований)

- ✅ Входить у стандартну бібліотеку Python 3.3+
- ✅ Простий у використанні
- ❌ Базовий функціонал
- ❌ Ручне керування залежностями

```bash
python -m venv my_env
source my_env/bin/activate
pip install requests
```

### Pipenv

- ✅ Автоматичне керування віртуальними середовищами
- ✅ Файл Pipfile замість requirements.txt
- ✅ Pipfile.lock для детермінованих збірок
- ✅ Розділення dev/prod залежностей
- ❌ Повільніший за pip

```bash
pipenv install requests
pipenv install --dev pytest
```

### Poetry

- ✅ Повнофункціональний менеджер проектів
- ✅ Сучасний формат pyproject.toml (PEP 518)
- ✅ Управління версіями та публікацією пакетів
- ✅ Швидкий розв'язувач залежностей
- ✅ Групи залежностей (dev, docs, test)
- ❌ Додаткове встановлення

```bash
poetry add requests
poetry add --group dev pytest
```

## Робочий процес

### Типовий процес роботи

1. **Створення** віртуального середовища
2. **Активація** середовища
3. **Встановлення** залежностей
4. **Розробка** проекту
5. **Фіксація** залежностей у файл
6. **Деактивація** середовища

### Файли залежностей

| Інструмент | Файл залежностей | Файл lock |
|-----------|------------------|-----------|
| venv/pip  | requirements.txt | - |
| Pipenv    | Pipfile          | Pipfile.lock |
| Poetry    | pyproject.toml   | poetry.lock |

## Важливі поняття

### PyPI (Python Package Index)

Офіційний репозиторій Python пакетів - [pypi.org](https://pypi.org)

### pip

Стандартний менеджер пакетів Python для встановлення з PyPI:

```bash
pip install package_name
pip uninstall package_name
pip list
pip freeze
```

### requirements.txt

Текстовий файл зі списком залежностей:

```txt
Flask==3.1.3
requests==2.32.5
jikanpy-v4==1.0.2
```

Створення:
```bash
pip freeze > requirements.txt
```

Встановлення:
```bash
pip install -r requirements.txt
```

## Коли що використовувати?

### venv - для простих проектів
- Навчальні проекти
- Швидкі експерименти
- Коли не потрібна складна логіка залежностей

### Pipenv - для веб-додатків
- Django/Flask проекти
- Коли потрібні dev-залежності
- Командна розробка

### Poetry - для бібліотек та складних проектів
- Створення власних пакетів
- Публікація на PyPI
- Проекти з багатьма групами залежностей
- Монорепозиторії
